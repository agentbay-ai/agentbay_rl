## 第一部分：算法理论基础

---

## 1. 算法简介与研究背景

### 1.1 核心研究问题

深度确定性策略梯度（Deep Deterministic Policy Gradient, DDPG）算法作为解决**连续动作空间**强化学习问题的关键方法，其核心研究问题在于：**如何在高维连续控制环境中实现稳定、高效的策略学习**。

传统深度Q网络（DQN）在离散动作空间中表现优异，但面对连续动作时存在根本性局限——难以通过神经网络输出无限可能的动作值并从中选择最优解。DDPG通过融合确定性策略梯度（DPG）理论与DQN的工程技巧，构建了Actor-Critic架构，有效克服了这一挑战。

### 1.2 DDPG的基本原理

DDPG是一种**无模型、离策略（off-policy）的Actor-Critic算法**，其核心在于学习一个确定性策略 `μ(s|θ)`，该策略直接将状态 `s` 映射到具体动作 `a`，而非概率分布。

**策略网络（Actor）** 的目标是最大化期望累积回报：
```
J(θ) = E[Q(s, μ(s))]
```

其更新梯度由评论家网络（Critic）提供的Q值梯度驱动：
```
∇_θ J ≈ E[∇_a Q(s,a)|_{a=μ(s)} · ∇_θ μ(s)]
```

这种设计避免了在连续空间中对动作进行采样和搜索，显著提升了计算效率。

### 1.3 核心特性概览

| 特性 | DDPG | DQN | PPO | TD3 |
|------|------|-----|-----|-----|
| **动作空间类型** | 连续 | 离散 | 连续/离散 | 连续 |
| **策略类型** | 确定性策略 | 值函数 | 随机策略 | 确定性策略 |
| **核心机制** | Actor-Critic、目标网络、OU噪声 | 经验回放、目标网络 | 信任区域优化、裁剪机制 | 双Q网络、延迟更新 |
| **主要优势** | 专为连续控制设计，端到端学习 | 在离散任务中稳定高效 | 样本效率高、训练稳定 | 改进DDPG的过估计问题 |
| **主要局限** | 训练不稳定，对超参数敏感 | 无法处理连续动作空间 | 在高维连续控制中可能不够精准 | 计算开销略大 |

DDPG借鉴了DQN的**经验回放缓冲区**和**目标网络**技术以稳定训练，并通过引入**奥恩斯坦-乌伦贝克（Ornstein-Uhlenbeck）过程**生成的探索噪声来增强在连续空间中的探索能力。

## 2. 算法起源与发展脉络

### 2.1 理论根基：两个关键里程碑

DDPG的理论根基建立在两个关键里程碑之上：

1. **确定性策略梯度定理（DPG, 2014）**
   - Silver等人首次从数学上严格证明了在无模型设定下，确定性策略梯度的存在性与可计算性
   - 为后续研究奠定了理论基础

2. **深度Q网络（DQN, 2013-2015）**
   - 成功将深度神经网络引入Q学习
   - 提出了经验回放和目标网络等稳定化技术

### 2.2 算法演进路径

```
Policy Gradient (PG, 2000)
    │
    ▼
Deterministic Policy Gradient (DPG, 2014)
    │                                    Deep Q-Networks (DQN, 2013-2015)
    │                                              │
    └──────────────────┬───────────────────────────┘
                       ▼
        Deep Deterministic Policy Gradient (DDPG, 2015)
                       │
           ┌───────────┼───────────┐
           ▼           ▼           ▼
         TD3        SAC         PPO
       (2018)     (2018)      (2017)
```

| 算法 | 提出年份 | 核心贡献 | 动作空间类型 | 关键技术 |
|------|----------|----------|--------------|----------|
| Policy Gradient (PG) | 2000 | 直接优化策略参数 | 连续/离散 | 策略梯度定理 |
| DPG | 2014 | 证明确定性策略梯度存在 | 连续 | 确定性策略、Actor-Critic |
| DQN | 2013-2015 | 深度神经网络用于Q学习 | 离散 | 经验回放、目标网络 |
| **DDPG** | **2015** | **将DQN思想融入DPG** | **连续** | **深度Actor-Critic、经验回放、目标网络** |
| PPO | 2017 | 稳定的策略优化 | 连续/离散 | 信任区域、裁剪机制 |
| TD3 | 2018 | 改进DDPG稳定性 | 连续 | 双Critic、延迟更新 |

### 2.3 DDPG的历史定位

DDPG并非凭空出现，而是强化学习领域内思想不断交叉融合的必然产物：
- 首先继承了PG方法**直接学习策略**的优势
- 然后通过DPG定理将其限定于更高效的**确定性策略**范畴
- 最后借助DQN在处理高维感知输入方面的突破，利用深度学习强大的函数逼近能力

自2015年提出以来，DDPG迅速成为解决机器人控制、自动驾驶等连续控制任务的基准算法之一，并启发了TD3、SAC等一系列更先进的后继算法。

## 3. Actor-Critic 架构详解

### 3.1 双网络协同机制

Actor-Critic架构通过将**策略学习（Actor）** 与**价值评估（Critic）** 解耦，有效提升了在连续动作空间中的学习效率与稳定性。

```
┌─────────────────────────────────────────────────────────────────┐
│                    DDPG 四网络架构                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│    ┌─────────────┐              ┌─────────────┐                 │
│    │   Actor     │              │   Critic    │                 │
│    │  (主网络)   │              │  (主网络)   │                 │
│    │  μ(s|θ^μ)  │──动作 a──▶  │ Q(s,a|θ^Q) │                 │
│    └─────────────┘              └─────────────┘                 │
│           │                            │                        │
│     软更新 τ                      软更新 τ                      │
│           ▼                            ▼                        │
│    ┌─────────────┐              ┌─────────────┐                 │
│    │   Actor     │              │   Critic    │                 │
│    │ (目标网络)  │              │ (目标网络)  │                 │
│    │  μ'(s|θ')  │              │  Q'(s,a|θ') │                 │
│    └─────────────┘              └─────────────┘                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**Actor网络**：
- 负责学习确定性策略 `μ_θ(s)`
- 直接从状态 `s` 映射到具体动作 `a`
- 避免了随机策略中对动作空间的积分运算
- 显著降低计算复杂度和策略梯度估计的方差

**Critic网络**：
- 学习状态-动作价值函数 `Q^μ(s, a)`
- 评估在状态 `s` 下执行动作 `a` 的长期回报
- 为Actor提供梯度信号

### 3.2 Critic网络更新

Critic网络的更新目标为最小化时间差分（TD）误差：

```
L(φ) = E[(r + γ·Q'(s', μ'(s')) - Q(s, a))²]
```

其中：
- `Q'` 和 `μ'` 分别是Critic和Actor的目标网络
- `γ` 是折扣因子
- 期望值从经验回放缓冲区中采样计算

### 3.3 Actor网络更新

Actor网络的性能目标函数为：
```
J(θ^μ) = E[Q(s, μ(s|θ^μ))]
```

根据确定性策略梯度定理，策略梯度可展开为：
```
∇_{θ^μ} J ≈ (1/N) Σ ∇_a Q(s, a)|_{a=μ(s)} · ∇_{θ^μ} μ(s)
```

这表明Actor的更新依赖于Critic网络对动作价值的梯度信息，体现了Actor-Critic框架的**"Critic指导Actor"** 的闭环优化机制。

### 3.4 与其他架构对比

| 对比维度 | DQN | DDPG | TD3 | PPO |
|---------|-----|------|-----|-----|
| 动作空间类型 | 离散 | 连续 | 连续 | 连续/离散 |
| 算法类型 | 基于值函数 | 基于策略+价值 | 基于策略+价值 | 基于策略 |
| 网络数量 | 2个 | **4个** | 6个 | 2个 |
| 动作选择机制 | argmax Q(s,a) | **策略网络直接输出** | 策略网络直接输出 | 采样策略分布 |
| Critic数量 | 1 | 1 | **2** | 1（可选） |
| 目标网络 | 是 | **是** | 是 | 否 |

## 4. 关键技术创新

DDPG算法的稳定性与效率在很大程度上依赖于**经验回放**与**目标网络**这两项关键技术。

### 4.1 经验回放机制

经验回放通过打破智能体与环境交互数据的时间相关性，使得训练样本近似满足独立同分布假设。

**标准经验回放**：
- 缓冲区容量通常设为 10⁵～10⁶
- 训练时随机采样 64~256 条经验进行小批量更新
- 显著提升样本利用率
- 避免连续样本导致的参数震荡

**改进方案**：

| 改进方法 | 核心思想 | 优势 |
|----------|----------|------|
| **优先经验回放（PER）** | 根据TD误差动态分配采样优先级 | 加快收敛速度 |
| **情节经验回放（EER）** | 按episode组织，优先采样高回报样本 | 提升策略学习质量 |
| **双缓冲区** | 分割缓冲区，动态自适应采样 | 优化回放效率 |

### 4.2 目标网络与软更新

DDPG为Actor和Critic分别构建了**目标网络**，形成四网络架构。目标网络通过**Polyak平均（软更新）** 缓慢追踪主网络参数：

```
θ' ← τ·θ + (1 - τ)·θ'
```

其中 `τ` 通常取 0.001～0.01，确保目标值变化平滑，避免训练发散。

**软更新的作用**：
- 目标网络的参数并非直接复制主网络
- 缓慢追踪使得学习目标更加稳定
- 解耦了策略更新与价值估计之间的强耦合
- 大幅提升训练稳定性

### 4.3 探索噪声

由于DDPG采用确定性策略，在每个状态下输出的动作是固定的，天然缺乏探索能力。因此必须通过在动作上叠加噪声来驱动探索。

**Ornstein-Uhlenbeck（OU）噪声**：
- 具有时间相关性
- 产生更平滑的探索轨迹
- 特别适用于物理模拟等需要连续控制的场景

**高斯噪声**：
- 简单的零均值高斯噪声
- 在多数任务中同样有效
- 实现更简洁，避免OU过程的超参数调优负担

**噪声参数设置建议**：
- 初始噪声标准差设置为动作空间上限的 10%-20%
- 采用噪声衰减策略：随训练进行逐渐减小噪声幅度
- 提升后期策略稳定性

### 4.4 四大技术支柱总结

| 技术 | 作用 | 关键参数 |
|------|------|----------|
| **经验回放** | 打破样本时间相关性，提升数据利用效率 | buffer_size: 10⁵~10⁶ |
| **目标网络** | 提供稳定的TD目标，防止价值函数震荡 | - |
| **软更新** | 确保目标网络参数缓慢变化，增强训练鲁棒性 | τ: 0.001~0.01 |
| **探索噪声** | 平衡探索与利用 | σ: 动作范围的10%-20% |

## 5. 数学推导与损失函数

### 5.1 Critic网络损失函数

Critic网络的目标是准确估计状态-动作对 `(s, a)` 的真实动作价值函数 `Q(s, a)`。其损失函数采用均方误差（MSE）形式：

```
L(θ^Q) = E[(Q(s, a|θ^Q) - y)²]
```

其中目标Q值 `y` 由目标网络计算：
```
y = r + γ·Q'(s', μ'(s'|θ^{μ'})|θ^{Q'})
```

- `Q'` 和 `μ'` 分别是Critic和Actor的目标网络
- `γ` 是折扣因子（通常 0.95~0.99）

### 5.2 Actor网络损失函数

Actor网络的目标是找到最优的确定性策略，使期望累积回报最大化。性能目标函数为：

```
J(θ^μ) = E[Q(s, μ(s|θ^μ)|θ^Q)]
```

为最大化 `J(θ^μ)`，Actor网络的损失函数设计为其负值：
```
L(θ^μ) = -E[Q(s, μ(s|θ^μ)|θ^Q)]
```

### 5.3 策略梯度推导

根据链式法则，策略梯度可展开为：

```
∇_{θ^μ} J ≈ (1/N) Σᵢ ∇_a Q(s, a|θ^Q)|_{s=sᵢ, a=μ(sᵢ)} · ∇_{θ^μ} μ(s|θ^μ)|_{s=sᵢ}
```

这表明：
- Actor的更新方向由Critic对动作价值的梯度 `∇_a Q` 引导
- 体现了Actor-Critic框架的紧密耦合
- Critic为Actor提供了"应该往哪个方向调整动作"的信号

### 5.4 完整算法流程

```
算法: DDPG
─────────────────────────────────────────────────────────────────
输入: 学习率 α_actor, α_critic, 软更新系数 τ, 折扣因子 γ
─────────────────────────────────────────────────────────────────
1. 初始化:
   - 初始化 Critic 网络 Q(s,a|θ^Q) 和 Actor 网络 μ(s|θ^μ)
   - 初始化目标网络: θ^{Q'} ← θ^Q, θ^{μ'} ← θ^μ
   - 初始化经验回放缓冲区 R

2. for episode = 1 to M do:
   - 初始化随机过程 N 用于动作探索
   - 获得初始观测状态 s₁
   
   3. for t = 1 to T do:
      - 根据当前策略和噪声选择动作: aₜ = μ(sₜ|θ^μ) + Nₜ
      - 执行动作 aₜ, 观察奖励 rₜ 和新状态 s_{t+1}
      - 将转移 (sₜ, aₜ, rₜ, s_{t+1}) 存储到 R
      
      - 从 R 中随机采样一批 N 个转移 (sᵢ, aᵢ, rᵢ, s_{i+1})
      - 计算目标值: yᵢ = rᵢ + γ·Q'(s_{i+1}, μ'(s_{i+1}|θ^{μ'})|θ^{Q'})
      
      - 更新 Critic: 最小化 L = (1/N)Σ(yᵢ - Q(sᵢ, aᵢ|θ^Q))²
      - 更新 Actor: 使用采样的策略梯度
      
      - 软更新目标网络:
        θ^{Q'} ← τθ^Q + (1-τ)θ^{Q'}
        θ^{μ'} ← τθ^μ + (1-τ)θ^{μ'}

4. end for
─────────────────────────────────────────────────────────────────
```

## 6. 实验任务：FetchReach 机械臂到达控制

本教学平台以 **FetchReach-v4** 机械臂控制任务作为 DDPG 算法的实践环境。该环境基于 MuJoCo 物理引擎构建，提供逼真的机器人仿真。

### 6.1 任务目标

控制 Fetch 机器人的七自由度机械臂，使其**末端执行器（夹爪）精确到达三维空间中的随机目标位置**。

```
┌────────────────────────────────────────────────────────────────┐
│                      FetchReach 任务示意                         │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│    🤖 Fetch 机器人                                              │
│    ┌─────┐                                                     │
│    │     │ ← 机械臂基座                                         │
│    └──┬──┘                                                     │
│       │                                                        │
│       ├──○──○──○ ← 七自由度关节                                │
│       │         \                                              │
│       │          ⬦ ← 末端执行器（夹爪）                         │
│       │              ╲                                         │
│       │               ╲  距离 d                                 │
│       │                ╲                                       │
│       │                 🎯 ← 目标位置 (desired_goal)            │
│                                                                │
│    ✅ 成功条件：d < 0.05 米                                     │
└────────────────────────────────────────────────────────────────┘
```

### 6.2 环境交互数据详解

FetchReach-v4 环境在每次交互中涉及以下关键数据：

#### 观测空间（Observation）

| 维度 | 数据内容 | 物理含义 |
|------|----------|----------|
| 0-2 | 夹爪位置 (x, y, z) | 末端执行器在笛卡尔坐标系中的三维位置 |
| 3-5 | 夹爪速度 (vx, vy, vz) | 末端执行器的线速度 |
| 6-7 | 夹爪手指位置 | 两个手指的开合状态 |
| 8-9 | 夹爪手指速度 | 手指的运动速度 |

**总维度**：10 维连续向量

#### 目标信息（Goal）

| 字段 | 维度 | 含义 |
|------|------|------|
| `desired_goal` | 3 维 | 需要到达的目标位置 (x, y, z) |
| `achieved_goal` | 3 维 | 当前末端执行器实际位置 (x, y, z) |

**状态构建**：网络输入 = `concat([observation, desired_goal])` = 13 维

#### 动作空间（Action）

| 维度 | 控制量 | 取值范围 |
|------|--------|----------|
| 0 | X 方向位移增量 | [-1, 1] |
| 1 | Y 方向位移增量 | [-1, 1] |
| 2 | Z 方向位移增量 | [-1, 1] |
| 3 | 夹爪开合控制 | [-1, 1] |

**总维度**：4 维连续动作

#### 奖励机制（Reward）

**Dense Reward（密集奖励）**：
```
reward = -distance(achieved_goal, desired_goal)
```

- 奖励值为**负的欧几里得距离**
- 距离越近，奖励越接近 0
- 完美到达目标时奖励 = 0

#### 终止条件

| 条件类型 | 说明 |
|----------|------|
| `truncated` | 达到最大步数限制（50 步），episode 被截断 |
| `terminated` | FetchReach 始终为 False（无真实终止条件） |
| `is_success` | 当距离 < 0.05 米时标记为成功 |

### 6.3 任务挑战分析

| 挑战 | 描述 | DDPG 应对策略 |
|------|------|---------------|
| **连续动作空间** | 4 维连续控制量，无法枚举 | 确定性策略直接输出动作值 |
| **精确定位要求** | 成功阈值仅 5 厘米 | HER 提升稀疏场景样本效率 |
| **高维状态空间** | 13 维输入（含目标） | 深度网络强函数逼近能力 |
| **物理动力学** | 需考虑惯性、摩擦力 | 经验回放学习动力学规律 |

## 7. 多沙箱并行训练框架

本平台采用**本地训练 + 云端沙箱执行**的分布式架构，实现高效的强化学习训练。

### 7.1 整体架构概览

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          本地训练端（用户浏览器/服务器）                    │
```

## 9. 神经网络结构

### 9.1 Actor 网络（策略网络）

```python
class Actor(nn.Module):
    """确定性策略网络：状态 → 动作"""
    def __init__(self, state_dim=13, action_dim=4, max_action=1.0):
        super(Actor, self).__init__()
        # 两层隐藏层，每层 256 个神经元
        self.l1 = nn.Linear(state_dim, 256)
        self.l2 = nn.Linear(256, 256)
        self.l3 = nn.Linear(256, action_dim)
        self.max_action = max_action

    def forward(self, state):
        a = torch.relu(self.l1(state))
        a = torch.relu(self.l2(a))
        # tanh 将输出限制在 [-1, 1]，再缩放到动作空间
        return self.max_action * torch.tanh(self.l3(a))
```

**网络结构示意**：
```
输入层 (13维)     隐藏层1 (256)    隐藏层2 (256)    输出层 (4维)
┌───────────┐    ┌───────────┐    ┌───────────┐    ┌───────────┐
│ observation│───►│   ReLU    │───►│   ReLU    │───►│   tanh    │
│ + goal    │    │           │    │           │    │  × max    │
│  (13维)   │    │  (256)    │    │  (256)    │    │   (4维)   │
└───────────┘    └───────────┘    └───────────┘    └───────────┘
```

### 9.2 Critic 网络（价值网络）

```python
class Critic(nn.Module):
    """动作价值函数：(状态, 动作) → Q值"""
    def __init__(self, state_dim=13, action_dim=4):
        super(Critic, self).__init__()
        # 状态和动作拼接后输入
        self.l1 = nn.Linear(state_dim + action_dim, 256)
        self.l2 = nn.Linear(256, 256)
        self.l3 = nn.Linear(256, 1)

    def forward(self, state, action):
        # 拼接状态和动作
        sa = torch.cat([state, action], dim=1)
        q = torch.relu(self.l1(sa))
        q = torch.relu(self.l2(q))
        return self.l3(q)
```

**网络结构示意**：
```
输入: state(13) + action(4) = 17维
┌───────────────┐    ┌───────────┐    ┌───────────┐    ┌───────┐
│ concat(s, a)  │───►│   ReLU    │───►│   ReLU    │───►│ Linear│
│    (17维)     │    │  (256)    │    │  (256)    │    │ (1维) │
└───────────────┘    └───────────┘    └───────────┘    └───────┘
                                                           │
                                                           ▼
                                                        Q(s, a)
```

### 9.3 四网络架构

DDPG 使用**主网络 + 目标网络**的双网络结构：

| 网络 | 作用 | 更新方式 |
|------|------|----------|
| Actor 主网络 | 选择动作 | 梯度上升（最大化 Q） |
| Actor 目标网络 | 计算 TD 目标中的下一动作 | 软更新（τ=0.005） |
| Critic 主网络 | 评估 Q 值 | 梯度下降（最小化 TD 误差） |
| Critic 目标网络 | 计算 TD 目标 | 软更新（τ=0.005） |

## 10. HER 重标记实现

### 10.1 核心代码

```python
class HERReplayBuffer:
    def process_episode(self, episode_transitions, worker_id=-1):
        """处理单个 episode，执行 HER 目标重标记"""
        episode_length = len(episode_transitions)
        
        # 1. 添加原始经验（使用 terminated 而非 truncated 作为 done）
        for t in episode_transitions:
            done_for_td = t.get('terminated', False)  # FetchReach 始终 False
            self.buffer.append((
                t['state'], t['action'], t['reward'],
                t['next_state'], done_for_td
            ))
        
        # 2. HER 重标记：为每个 transition 生成虚拟经验
        for idx, transition in enumerate(episode_transitions):
            # FUTURE 策略：从后续步骤采样目标
            future_indices = list(range(idx, episode_length))
            selected = random.sample(future_indices, 
                                    min(self.n_sampled_goal, len(future_indices)))
            
            for future_idx in selected:
                # 用 future 时刻的 achieved_goal 作为新 desired_goal
                new_goal = episode_transitions[future_idx]['achieved_goal']
                
                # 重新计算奖励
                new_reward = -np.linalg.norm(
                    transition['next_achieved_goal'] - new_goal
                )
                
                # 重建状态：替换末尾的 goal 部分
                obs_dim = len(transition['state']) - self.goal_dim
                her_state = np.concatenate([
                    transition['state'][:obs_dim], new_goal
                ])
                her_next_state = np.concatenate([
                    transition['next_state'][:obs_dim], new_goal
                ])
                
                # 添加虚拟经验
                self.buffer.append((
                    her_state, transition['action'], new_reward,
                    her_next_state, done_for_td
                ))
```

### 10.2 HER 效果示意


## 11. 训练参数详解

### 11.1 核心超参数

| 参数 | 默认值 | 含义 | 调参建议 |
|------|--------|------|----------|
| `learning_rate` | 1e-3 | Actor/Critic 学习率 | 过大震荡，过小收敛慢 |
| `gamma` | 0.99 | 折扣因子 | 越大越重视长期回报 |
| `tau` | 0.005 | 软更新系数 | 越小目标网络越稳定 |
| `batch_size` | 256 | 每次采样批量大小 | 越大方差越小 |
| `buffer_size` | 1,000,000 | 经验回放容量 | 需足够大保证多样性 |

### 11.2 HER 相关参数

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `n_sampled_goal` | 8 | 每个 transition 重标记的目标数 |
| `goal_dim` | 3 | 目标向量维度（xyz） |
| `distance_threshold` | 0.05 | 成功判定距离阈值 |

### 11.3 训练流程参数

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `episodes` | 2000 | 总训练轮数 |
| `updates_per_episode` | 100 | 每轮梯度更新次数 |
| `warmup_rounds` | 10 | 预热轮数（只收集不更新） |
| `noise_start` | 0.3 | 初始探索噪声标准差 |
| `noise_end` | 0.02 | 最终探索噪声标准差 |
| `log_interval` | 10 | 日志输出和演示间隔 |

### 11.4 噪声衰减策略

```python
def _get_noise_scale(self, episode_idx):
    """分段线性衰减：前30%保持高探索，之后线性降到 noise_end"""
    warmup_frac = 0.3
    if episode_idx < self.episodes * warmup_frac:
        return self.noise_start  # 前30%保持0.3
    
    # 之后线性衰减
    decay = (episode_idx - self.episodes * warmup_frac) / \
            (self.episodes * (1 - warmup_frac))
    return max(self.noise_end, 
               self.noise_start - (self.noise_start - self.noise_end) * decay)
```

## 12. 常见问题与调优

### 12.1 训练不收敛

**症状**：成功率长期停留在 10% 以下

**可能原因与解决方案**：
| 原因 | 解决方案 |
|------|----------|
| 噪声过小 | 增大 `noise_start` 到 0.4-0.5 |
| 预热不足 | 增加 `warmup_rounds` 到 20-30 |
| 学习率过大 | 降低到 1e-4 |
| HER 采样不足 | 增加 `n_sampled_goal` 到 16 |

### 12.2 训练震荡

**症状**：成功率时高时低，不稳定

**解决方案**：
- 减小学习率
- 减小 `tau` 值（如 0.001）
- 增大 `batch_size`（如 512）

### 12.3 沙箱通信超时

**症状**：日志显示 "batch_step 失败" 或 "reset 超时"

**解决方案**：
- 检查网络连接稳定性
- 减少 `parallel_workers` 数量
- 增加通信轮询间隔

### 12.4 显存/内存不足

**症状**：训练中途 OOM 崩溃

**解决方案**：
- 减小 `buffer_size`（如 500,000）
- 减小 `batch_size`（如 128）
- 减少并行沙箱数
