# DDPG 机械臂抓取训练项目

## 项目概述

本项目实现了基于沙箱隔离架构的深度确定性策略梯度（DDPG）算法训练系统，通过将环境执行与训练算法分离，在AgentBay沙箱中运行机械臂环境，本地执行训练算法，实现了安全、高效的强化学习训练。

### 核心架构特点
- **分离式计算架构**: 环境执行在沙箱中，训练算法在本地运行
- **实时可视化**: 沙箱提供机械臂实时渲染展示
- **双沙箱设计**: 训练沙箱 + 测试沙箱，便于对比验证
- **Web前端监控**: 通过WebSocket实时展示训练进度和沙箱状态

## 核心组件详解

### 1. 沙箱DDPG运行器 (`sandbox_ddpg.py`) - 核心协调组件
**职责**: 整个训练流程的协调和管理
- **双沙箱创建**: 创建训练沙箱和测试沙箱
- **依赖安装**: 在沙箱中安装gymnasium-robotics等必要依赖
- **环境部署**: 将环境执行器部署到训练沙箱
- **训练启动**: 启动本地训练算法并连接沙箱环境
- **流程协调**: 管理训练-测试-保存的完整流程

### 2. 沙箱环境执行器 (`sandbox_env_executor.py`) - 沙箱环境核心
**职责**: 在沙箱中运行真实的gym环境
- **环境初始化**: 创建FetchReachDense-v4环境（支持渲染模式）
- **命令处理**: 接收来自本地训练算法的动作指令
- **状态反馈**: 返回观测、奖励、完成状态等信息
- **实时渲染**: 在沙箱中展示机械臂执行过程
- **进程管理**: 独立进程运行，通过文件通信交互

### 3. 本地DDPG训练代码 (`local_ddpg_training_codes.py`) - 训练算法核心
**职责**: 在本地执行DDPG训练算法
- **算法实现**: 基于Stable-Baselines3的DDPG+HER算法
- **环境交互**: 通过MCP协议与沙箱环境通信
- **训练循环**: 执行完整的训练流程
- **模型管理**: 保存检查点和最终模型
- **进度跟踪**: 记录训练进度和性能指标

### 4. 本地监控GUI代码 (`local_ddpg_monitor_codes.py`) - 可视化组件
**职责**: 提供训练进度的图形化监控
- **GUI界面**: 实时显示训练状态和性能指标
- **数据可视化**: 图表展示奖励、成功率等关键指标
- **进度监控**: 训练进度条和实时统计

### 5. 独立训练脚本 (`improved_ddpg_training.py`) - 测试组件
**职责**: 独立的DDPG训练实现（用于对比测试）
- **完整训练流程**: 不依赖沙箱的独立训练
- **性能基准**: 用于验证沙箱架构的有效性
- **调试工具**: 便于算法调试和参数调优

### 2. 环境支持
- **FetchReachDense-v4**: 机械臂末端执行器到达指定目标位置任务（密集奖励版本）
- **观测空间**: goal-based observation（包含达成目标、当前目标、机器人状态）
- **动作空间**: 4维连续向量（3维控制末端移动 + 1维控制夹爪）
- **奖励机制**: 基于距离的密集奖励 + 任务完成奖励

## 训练流程详解

### 1. 系统启动流程
1. **前端选择**: 用户在Web界面选择DDPG算法
2. **沙箱创建**: 系统创建训练沙箱和测试沙箱
3. **环境部署**: 在训练沙箱中部署环境执行器
4. **依赖安装**: 安装gymnasium-robotics等必要依赖
5. **环境启动**: 启动沙箱中的环境执行器进程
6. **训练开始**: 本地启动DDPG训练算法

### 2. 训练执行流程
```
本地训练算法 ←→ MCP通信协议 ←→ 沙箱环境执行器
     ↓                                ↓
  动作决策                         环境执行
     ↓                                ↓
  策略更新                         状态反馈
     ↓                                ↓
  模型保存                         渲染展示
```

### 3. 实时交互机制
- **动作传递**: 本地算法通过MCP发送动作指令到沙箱
- **状态获取**: 沙箱环境执行动作并返回观测、奖励等信息
- **渲染展示**: 沙箱中实时渲染机械臂执行过程
- **进度同步**: 训练进度通过WebSocket实时推送到前端

### 4. 模型管理流程
1. **定期保存**: 训练过程中定期保存模型检查点
2. **性能评估**: 使用测试沙箱评估模型性能
3. **最佳模型**: 保存性能最优的模型版本
4. **最终模型**: 训练结束时保存最终模型

## 常见问题解答

### Q: 为什么在沙箱中看不到机械臂渲染？
A: 确保环境执行器初始化时设置了`render_mode="human"`，并且沙箱支持图形界面显示。

### Q: 训练速度慢怎么办？
A: 可以调整评估频率、并行环境数量等参数，或在非渲染模式下进行训练。

### Q: 沙箱连接失败如何处理？
A: 检查AgentBay API密钥、网络连接，以及沙箱资源是否充足。

### Q: 如何监控训练进度？
A: 通过前端界面的进度条、性能图表和实时日志进行监控。
